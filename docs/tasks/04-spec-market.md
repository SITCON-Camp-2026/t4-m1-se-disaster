# Spec 市集

## 時間

13:00–13:30

## 你現在拿到的來源

- 自己組的 `docs/spec.md`
- 自己組的 `docs/data-contract.md`
- 其他組的 spec
- mentor / 同學質疑

## 你要做什麼

閱讀其他組的 spec，針對工程風險提出具體質疑。

你可以質疑：

1. 使用者不明
2. 資料來源不明
3. 狀態不可驗證
4. 責任角色不明
5. scope 太大
6. AI 權限過大
7. 無法用前端 demo 呈現
8. schema input / output 不清楚

收到質疑後，請回頭修正自己的 spec。

## 你不需要做什麼

- 不評比誰的點子比較酷
- 不幫別組寫解法
- 不擴大自己的 scope
- 不把質疑變成辯論輸贏

## 可以怎麼使用 Coding Agent

可以請 Coding Agent 幫你檢查 spec 是否有缺漏，但最後質疑與取捨由小組決定。

## 必須交付什麼

- [ ] 對其他組提出至少 2 個具體質疑
- [ ] 自己收到質疑後，更新 `docs/spec.md`
- [ ] 在 `docs/decisions.md` 記錄至少 1 個取捨

## 完成定義

你們的 spec 被質疑後仍能說清楚：主流程是什麼、資料從哪裡來、哪些狀態需要人工確認、哪些功能今天不做。

## 停止條件

13:30 停止新增想法，準備進入 Scope Lock。
